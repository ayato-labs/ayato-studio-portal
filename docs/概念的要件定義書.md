# Ayato Studio Portal - 概念的要件定義書 (Conceptual Requirements)

## 1. プロジェクトの目的と概要
本プロジェクトは、自動知性生成エンジン（`ayato_reporter`）が収集・分析した最先端のAI論文や技術ニュースを、日本のエンジニアや意思決定者に届けるためのAI特化型ポータルサイト（`ayato-studio.ai`）です。
「運用・維持コストゼロ」「耐障害性極大」を追求し、Next.js (SSG) + Supabase + Cloudflare Pages による完全なサーバーレス・エッジ配信アーキテクチャを採用しています。

---

## 2. 実装する機能 (In-Scope Features)

「AIのすべてが日本語で一目でわかる」体験を提供するため、以下の3つのコアコンテンツに特化したシングルページ＋詳細ページの構成を実装します。

### 2-1. AIニュース速報掲示板 (Bulletin Board)
- **概要**: 直近に発生した国内外のAIや主要テック企業の動きを、超要約の箇条書き（Bulletin）でタイムライン表示。
- **目的**: 毎日の技術トレンドや企業動向を数秒でアグリゲートして把握可能にする。

### 2-2. 最新AI論文フィード (arXiv.org cs.AI)
- **概要**: arXivのAIカテゴリ（cs.AI）に投稿された最新論文から、AIエージェントがインパクトスコア（0-100）を算出し、高スコアの重要論文のみを日本語解説。
- **詳細閲覧**: 論文ごとの背景、新規性、技術的詳細、ビジネスインパクトをシンタックスハイライト付きのMarkdownで美しくレンダリング。

### 2-3. LLM性能・コスト統計ダッシュボード (LLM Stats)
- **概要**: Claude, GPT, Geminiなどの主要モデルのスペック（コンテキスト長、日本語ベンチマーク、100万トークンあたりコスト、推論速度）のインタラクティブな比較グラフと、ソート可能なスペックテーブル。
- **目的**: システムへLLM APIを組み込む際の、客観的な意思決定指標を提供する。

---

## 3. 実装しないと決めた機能 (Non-Goals)

システムをシンプルに保ち、運用負荷をゼロにするため、以下の機能は明示的にスコープ外（実装しない）とします。

- **ユーザー認証・ログイン (Auth)**: 一般公開のインテリジェンス情報であるため、セッション管理やユーザー単位のパーソナライズは一切行いません。
- **動的CMS・エディタ機能**: ポータルは完全に「閲覧専用（Read-Only）」とし、データの追加・更新はバックエンドの `ayato_reporter` からのSupabase書き込みと、それに伴う自動ビルドトリガー（自律デプロイ）のみで行います。
- **Node.jsサーバーサイド動的演算**: サーバー維持コストとセキュリティ脆弱性を排除するため、リクエスト時にHTMLを動的構築するSSRは採用せず、全ページを事前に静的生成（SSG/ISR）してエッジサーバーから即時返却します。

## 4. 設計・実装の美学 (Design & Architecture Philosophy)

### 4-1. プレミアム・アジリティ (Wow Aesthetics)
Webサイトはブランドの第一印象を決定づける最重要コンポーネントです。デフォルトの素朴なUIコンポーネントは排除し、深みのあるダークモード、滑らかなグラデーション、グラスモフィズム、洗練されたフォント（Google Fonts: Outfit / Inter）、および適度なホバーアニメーションを駆使し、アクセスした瞬間に「高度な知性」を感じさせるプレミアムな体験を構築します。

### 4-2. モバイルファースト・一点集中
ユーザーがスキマ時間にスマートフォンで情報を消費するユースケースを最優先とし、親指で操作しやすいナビゲーションや視認性の高いタイポグラフィを採用します。PCでの閲覧時は単に画面横いっぱいに引き伸ばすのではなく、可読性の高い最大幅（Max Width）を設定してコンテンツの美しさと視認性を担保します。

### 4-3. Row Level Security (RLS) による強固なデータ防衛
フロントエンドから直接 Supabase へアクセスしてデータを取得する構造上、アノニマスキー（Anon Key）がブラウザに露出します。これを悪用したデータ改ざんを防ぐため、DB側で RLS Policy を厳格に敷き、「一般クライアント（anonロール）からのリクエストは `SELECT` のみ許可し、`INSERT/UPDATE/DELETE` は完全に拒否する」防御を徹底します。
