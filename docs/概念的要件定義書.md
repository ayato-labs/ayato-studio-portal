# 要件定義 (コンセプトドキュメント)

## 1. プロジェクトの目的と概要
Ayato Reporter が生成するレポートの閲覧、および高度な資産運用戦略の策定を支援するポータルサイト (Ayato Studio Portal) を構築する。
「運用コストゼロ・手間ゼロ・プライバシー最大化」を突き詰めるため、サーバーレスアーキテクチャおよびクライアントサイド完結型の知能を統合する。

## 2. 実装する機能 (In-Scope Features)

### 2.1 Ayato Reporter (閲覧機能)
* **レポート一覧ページ (Feed)**: Supabase に蓄積された最新レポートをタイムライン表示。
* **レポート詳細ページ (Article)**: Markdown レポートを美しくレンダリング。

### 2.2 Portfolio Strategist (資産運用戦略)
* **アセットアロケーション管理**:
    * 個別銘柄のマイクロ管理を排し、アセットクラス（インデックス、株、債券、金、現金、暗号資産）ごとの「総額」によるハイレベル管理。
* **ノーセル・リバランス・シミュレーション**:
    * 売却による課税を避け、新規の「追加投資額」のみで目標配分に近づけるための最適解を算出。
* **カスタム戦略設定**:
    * ユーザーが自身の投資目標に合わせ、カテゴリごとのターゲット比率を自由にカスタマイズ可能。

## 3. 実装しないと決めた機能 (Non-Goals)
* **サーバーサイドでの個人データ保存**:
    * ポートフォリオデータ等は一切サーバーに送らず、ブラウザ内 (LocalStorage) で完結させる。
* **特定の投資銘柄の推奨 (投資助言の回避)**:
    * 具体的な個別銘柄の「買い」を指示するのではなく、カテゴリレベルでの「戦略との乖離」を提示するシミュレーターに徹する。
* **ユーザー認証 (Auth)**:
    * 認証コストを省き、ローカルストレージによる「匿名かつセキュア」な個人管理を採用。

## 4. 設計・実装の美学 (Design & Architecture Philosophy)

### 4.1 プレミアム・ライトテーマ (Wow Aesthetics)
* **思想**: 金融・リサーチツールには「清潔感」と「信頼性」が必要である。
* **方針**: 白背景を基調とし、グラスモフィズム、グラデーション、洗練されたタイポグラフィを用いた「プレミアム・ライトテーマ」を採用。

### 4.2 プライバシー・バイ・デザイン (Local-First)
* **思想**: 資産情報は究極の個人情報である。
* **方針**: 「サーバーにデータを送らない」ことを技術的制約とする。すべての計算と保存はユーザーのデバイス上で行われる。

### 4.3 規制への配慮 (Regulatory Compliance)
* **思想**: 技術は法律を尊重しなければならない。
* **方針**: 投資助言業に抵触しないよう、銘柄ごとの詳細な売買ガイドを避け、アセットクラス単位での「乖離シミュレーション」という枠組みを維持する。

## 5. 価値定量化エンジン (VQE)
「なんとなく便利」を「具体的なビジネス価値」へ変換するため、レポートの有用性評価やリサーチ効率向上時間を計測し、プロダクトの ROI を可視化する。
