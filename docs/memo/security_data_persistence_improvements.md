# セキュリティ・プライバシーとデータ永続化の改善案

## 課題・背景
*   現在、NISA Strategist や Portfolio などのアプリは、プライバシー保護とサーバーコスト削減の観点から「Zero-Data Architecture」を採用し、ユーザーデータをブラウザの Local Storage にのみ保存している。
*   しかし、Local Storage はユーザーがブラウザのキャッシュや履歴をクリアした際、あるいはシークレットモードで利用した際に容易に消失してしまう。
*   ユーザーが長期間かけて設定したポートフォリオなどのデータが不意に消えることは、深刻なUXの低下を招く。

## 改善策（アイデア）

### 1. JSON ファイルによるローカルデータのエクスポート／インポート機能
*   **内容**: ユーザーが現在の Local Storage に保存されている設定データ（JSON形式）を、物理的なファイルとしてダウンロード（エクスポート）できる機能を提供する。また、そのファイルを読み込んで復元（インポート）できる機能を追加する。
*   **メリット**:
    *   「データをクラウドサーバーに送らない（Ayato Studioが保持しない）」という強力なプライバシー保護の哲学を維持できる。
    *   ブラウザ変更時や端末の移行時にもデータを引き継ぐことが可能になる。
    *   ユーザー自身でデータのバックアップを管理できるため、不慮のデータ消失リスクに対する安心感を提供できる。
