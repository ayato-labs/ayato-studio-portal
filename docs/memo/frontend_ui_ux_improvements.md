# フロントエンド・UI/UXの改善案

## 課題・背景
*   現在のアプリ（NISA StrategistやPortfolio）の計算ロジックはブラウザのメインスレッド（UIスレッド）で実行されている。
*   将来的にモンテカルロ・シミュレーションなどの重い計算処理を追加した場合、低スペックなスマートフォン等で画面がフリーズする（UIがブロックされる）リスクがある。
*   現状でも美しいUIだが、画面遷移や状態変化時のアニメーションがCSS主導であり、さらにリッチな体験を提供する余地がある。

## 改善策（アイデア）

### 1. Web Workerによる計算処理のオフロード
*   **内容**: 重いシミュレーション計算をUIスレッドから切り離し、バックグラウンドのWeb Workerで実行するアーキテクチャを導入する。
*   **メリット**: 計算中もUIが滑らかに動き、ローディングアニメーションなどをカクつくことなく表示できる。UXの低下を防ぐ。

### 2. Framer Motionを用いたリッチなトランジション
*   **内容**: プロジェクトに既にインストールされている `framer-motion` をより積極的に活用する。
*   **具体例**:
    *   レポートのタブ切り替え時（Stock / Flow）の「Shared Element Transition（要素が滑らかに移動するアニメーション）」。
    *   計算結果グラフがマウントされる際の、より有機的なアニメーション。
*   **メリット**: 「Wow Aesthetics（プレミアム感）」というコンセプトをさらに強調し、ユーザーに高度な知性を感じさせる体験を提供する。
