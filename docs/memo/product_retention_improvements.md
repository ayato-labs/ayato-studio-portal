# プロダクト・リテンションの改善案

## 課題・背景
*   システムをシンプルに保つため、ユーザーアカウント登録や認証機能をあえて実装しない（捨てる）決断をしている。
*   そのため、ユーザーに直接アプローチする手段（メールマガジンやパーソナライズされた通知など）がなく、リテンション（再訪率）を高めるための能動的なフックが弱い。
*   新規コンテンツ（レポート等）が追加されても、ユーザーが自発的にサイトを訪れるか、外部SNSからの流入に依存する状態となっている。

## 改善策（アイデア）

### 1. 匿名性を保ったリテンション強化策の導入
アカウント機能を持たないという哲学を維持したまま、再訪を促す仕組みを構築する。

*   **アプローチ A: RSS / Atom フィードの提供**
    *   **内容**: 最新のレポート（Stock/Flow）を配信するRSSフィードの導線をUI上で強調し、ユーザーが普段使っているRSSリーダー（Feedly等）で購読できるようにする。
*   **アプローチ B: Web Push 通知 (PWA化) の検討**
    *   **内容**: VAPIDキーを用いた標準的な Web Push API を実装する。ユーザーがブラウザ上で「通知を許可」するだけで、個人情報を取得することなく、新しいレポート公開時などにプッシュ通知を送る仕組み。
*   **メリット**: アカウント作成というフリクション（摩擦）を生むことなく、継続的なユーザーとの接点（タッチポイント）を維持できる。
