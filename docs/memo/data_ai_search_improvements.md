# データ・AI・検索機能の改善案

## 課題・背景
*   AIが生成する大量の市場ニュース（Flow）と、人間が書いた洞察（Stock）が蓄積されていくシステム構造である。
*   現在は新しいもの順（タイムライン形式）での表示を主としており、外部の検索エンジン（Algolia等）の導入は見送っている。
*   今後データ量が膨大になった際、ユーザーが「過去の特定のトピックに関するインサイト」を探し出す手段がなく、価値あるデータが埋もれてしまう（死蔵される）リスクがある。

## 改善策（アイデア）

### 1. Supabase `pgvector` を用いたセマンティック検索（意味検索）
*   **内容**: 外部の検索SaaSを導入する代わりに、現在メインデータベースとして利用している Supabase の機能を拡張する。
    *   AIエージェント（Ayato Reporter等）がレポートを生成・DB保存する際、同時にそのテキストの Embedding（ベクトル化データ）を `pgvector` 拡張を用いて保存しておく。
    *   フロントエンドからユーザーが自然言語で検索した際、ベクトル類似度検索によって関連性の高いレポートを返す。
*   **メリット**:
    *   キーワードの完全一致ではなく、「意味・文脈」に基づいた高度な検索（Semantic Search）が可能になる。
    *   新たなインフラ（外部SaaS）を増やすことなく、既存の Supabase エコシステム内で完結するため、システムの複雑さと運用コストを抑えられる。
