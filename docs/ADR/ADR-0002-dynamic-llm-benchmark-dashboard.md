# ADR-0002: LLMベンチマーク＆コストダッシュボードの動的自動更新化とベンチマーク性能の動的推定ロジックの導入

- **Date**: 2026-06-09
- **Status**: Accepted
- **Deciders**: ayato-labs (Human), Antigravity (AI Agent)

## Context

当初、LLMベンチマークダッシュボードのモデル一覧および各ベンチマークスコア、APIコストデータは、ローカルの静的JSONファイル（`llm-stats.json`）にハードコードされていました。しかし、この構成には以下の問題が存在していました。

1. **情報の陳腐化**: LLM市場は非常にサイクルが早く、新しいモデルの登場や大幅な値下げが日常的に発生します。静的ハードコードでは短期間でラインナップが陳腐化し、古い情報を提供し続けることになります。
2. **高いメンテナンスコスト**: 最新モデルがリリースされるたびに、手動でモデル定義、OpenRouter等のモデルIDマッピング、各種独自ベンチマーク（日本語性能など）を調べてJSONを更新する必要があり、運用負荷が著しく高くなっていました。
3. **スタジオのブランド整合性**: 「AIネイティブシステム構築の専門技術スタジオ」を標榜しながら、表示データが2年前のような古い静的データで止まっていることは、技術的な信頼性を損なうリスクがありました。

他社サイトのスクレイピングは、クライアントサイドでのCORS制限やスクレイピング対象サイトの構造変化に対する脆弱性があるため、フロントエンド完結で信頼できるパブリックAPIから最新モデルデータを直接取得し、ダッシュボード全体を自律的かつ自動で再生成する仕組みが求められました。

## Decision

以下の通り、ダッシュボードアーキテクチャの動的化およびベンチマーク自動算出エンジンの導入を決定しました。

1. **OpenRouter APIによる最新モデルの動的フェッチ**:
   コンポーネント起動時に、OpenRouterのモデルエンドポイント (`https://openrouter.ai/api/v1/models`) を直接フェッチし、モデルのリリース状況およびAPI価格をリアルタイムに取得します。APIキーの認証は不要（Public API）です。

2. **最新主要モデルの自動フィルタリング**:
   全数百モデルの中から、主要ベンダー（openai, anthropic, google, meta-llama, qwen, deepseek, mistralai）の正規版モデルのみを自動抽出し、無料体験版（`:free` 等のサフィックス）やセーフティガードレール専用モデルを除外して、常に業界トップ10〜12の主要モデルにラインナップを動的整理します。

3. **Heuristic Engineによるベンチマーク＆速度の自動推定**:
   APIからは直接取得できない「日本語性能」「コーディング」「推論速度」「推論能力（reasoning）」について、APIのモデル単価（1MトークンあたりのUSDコスト＝性能のプロキシ）、コンテキスト長、モデル名の文字列シグナル（`coder`, `instruct`, `flash` 等）から、妥当な数値を動的に逆算・推定するロジックを実装しました。
   これにより、手動メンテナンスゼロで最新モデルのスコアが現実的な精度で自動追加されます。

4. **フォールバックとUI統合**:
   初期状態（またはAPI取得失敗時）はローカルのJSONデータ（`llm-stats.json`）を表示し、APIから動的データがフェッチされた瞬間に最新モデルリストへ滑らかに差し替えます。データソースの状態をユーザーに示すため、`Live Cost API` / `Static Cost` バッジをUIに組み込みました。

## Consequences

### Positive
- **完全な自動運転（メンテナンスフリー）**: 新モデルの追加や既存モデルの価格改定にフロントエンドが完全自動で追従します。
- **情報の最先端維持**: 2026年時点の最新の代表モデル（GPT-5.5, Claude 4.8, Gemini 3.5等）が発表とほぼ同時に即時リストに反映されます。
- **技術スタジオとしての信頼性向上**: 外部APIとヒューリスティック推定を組み合わせた高度なフロントエンド実装を公開することで、「AIネイティブな実装力」を自ら証明するアセットになります。

### Negative / Risks
- **推定スコアの正確性限界**: 実際のベンチマークを測定しているわけではないため、ヒューリスティックによる推定値（特に新モデルの日本語精度）が実際の能力とわずかに乖離する可能性があります。
  - **対策**: 将来的にベンチマークテスト結果を自動で測定・蓄積するバックエンドAPIを追加し、ヒューリスティック推定から「測定スコアAPI」へ動的にバインドを切り替えられる拡張性を維持します。
