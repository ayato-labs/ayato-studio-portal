# Ayato Studio Portal アーキテクチャ構成

本ドキュメントでは、Ayato Studio Portal のフロントエンド技術スタック、セキュリティ構造、および CI/CD デプロイメント・アーキテクチャについて詳述します。

---

## 1. システム・アーキテクチャ

本システムは、バックエンドの知性生成エンジン（`ayato_reporter`）でのデータ更新をトリガーに、GitHub Actions で静的生成（SSG）を行い、Cloudflare Pages へ静的アセットをデプロイする完全な GitOps および自律デプロイワークフローを採用しています。

```mermaid
graph TD
    Reporter[ayato_reporter (GCP Run Job)] -->|1. techデータ更新完了| GH_API[GitHub API (dispatch)]
    GH_API -->|2. イベント検知| GH_Workflow[GitHub Actions (Deploy Workflow)]
    GH_Workflow -->|3. Next.js ビルド (SSG)| CF_Pages[Cloudflare Pages]
    CF_Pages -->|4. エッジキャッシュ同期| User[ユーザー (ayato-studio.ai)]
    
    subgraph Client App
        User -->|5. 統計・詳細表示| App[Next.js 16 Portal]
        App -->|6. 読み取り専用クエリ| DB[(Supabase DB)]
    end
```

## 2. 技術スタック

ポータルサイトのフロントエンドは、パフォーマンス、SEO、および意匠性のすべてを最大化するモダンなスタックで構成されています。

| レイヤー | 技術 / ライブラリ | 役割 |
| :--- | :--- | :--- |
| **フレームワーク** | **Next.js 16 (App Router)** | 静的生成（SSG/ISR）による超高速ロードと、SEOメタデータの動的生成。 |
| **スタイリング** | **Tailwind 4 & Vanilla CSS** | ユーティリティファーストでの高速UIスタイリングと、カスタムCSSアニメーションの融合。 |
| **可視化エンジン** | **Recharts** | LLMのスペック（コスト、速度、性能）を視覚的に比較・分析するためのレスポンシブグラフ描画。 |
| **データベース連携** | **Supabase JS Client** | クライアントサイドからの直感的な非同期データフェッチ。 |

## 3. セキュリティとデータアクセス (DB防御)

ポータルサイトはバックエンドサーバーを仲介せず、クライアントサイドから直接 Supabase へクエリを発行します。これに伴う資格情報（Anon Key）の露出に対し、データベース層で厳格なセキュリティを適用しています。

- **Row Level Security (RLS) Policy**:
  - `generated_reports` などの公開データテーブルに対し、アノニマス（匿名）ロール（`anon`）からは **`SELECT`（読み取り）権限のみ** を許可。
  - `INSERT / UPDATE / DELETE`（書き込み・改ざん）操作は、GCPで動作する `ayato_reporter` のみが持つサービスロールキー（`service_role`）経由でのアクセス以外、データベースレベルで完全に遮断されます。

## 4. CI/CD および自律リリースフロー

### 4-1. 自動再構築チェーン
1. `ayato_reporter` でのAI・半導体（`tech`）データ収集が完了。
2. レポーターから GitHub API の `/repos/{owner}/{repo}/dispatches` に `rebuild-portal` イベントを送信。
3. GitHub Actions の `deploy.yml` ワークフローが起動し、Next.js アプリをビルド（データを取り込んで静的ページを出力）。
4. ビルド結果が Cloudflare Pages にデプロイされ、全世界のエッジノードにキャッシュが同期されます。

### 4-2. 品質管理
- **Ruff & ESLint**: PRおよびコミット時に、静的解析によるコード品質の検査を自動実行。
- **Type Check**: TypeScript の静的型検査（`tsc`）による実行時エラーの防止。
